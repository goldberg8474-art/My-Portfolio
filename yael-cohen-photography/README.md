# יעל כהן צלמת · Yael Cohen Photography

A production-ready bilingual (Hebrew/English) portfolio and booking-inquiry website for photographer Yael Cohen, built with Astro, TinaCMS, Resend, and Cloudinary. Deployed on Vercel.

---

## Technology Stack

| Technology | Version | Purpose |
|---|---|---|
| [Astro](https://astro.build) | `^5.18.0` | Static site framework with SSR API endpoint |
| [@astrojs/vercel](https://docs.astro.build/en/guides/integrations-guide/vercel/) | `^8.0.6` | Vercel adapter for static output + serverless function |
| [TinaCMS](https://tina.io) | `^2.8.2` | Git-backed headless CMS with visual editing |
| [@tinacms/cli](https://tina.io/docs/cli-overview/) | `^2.1.8` | TinaCMS dev server and build tooling |
| [Resend](https://resend.com) | `^6.9.3` | Transactional email API for contact form |
| [TypeScript](https://www.typescriptlang.org) | `^5.7.0` | Type safety throughout |
| [Cloudinary](https://cloudinary.com) | URL-only | Image delivery and transformation (no SDK) |

## Local Development Setup

### Prerequisites

- Node.js 18 or later
- A [Tina Cloud](https://app.tina.io) account (free tier works)
- A [Resend](https://resend.com) account with a verified sending domain
- A [Cloudinary](https://cloudinary.com) account (free tier works)

### Steps

1. **Clone the repository and install dependencies:**

   ```bash
   git clone <your-repo-url>
   cd yael-cohen-photography
   npm install
   ```

2. **Create your environment file:**

   ```bash
   cp .env.example .env
   ```

   Fill in all values in `.env` — see the [Environment Variables](#environment-variables) section below.

3. **Start the development server:**

   ```bash
   npm run dev
   ```

   This runs `tinacms dev -c 'astro dev'`, which starts both the TinaCMS local server and the Astro dev server simultaneously. The site is available at `http://localhost:4321` and the CMS admin at `http://localhost:4321/admin`.

---

## Build Command

```bash
npm run build
```

This runs `tinacms build && astro build` — first generating the Tina admin bundle into `public/admin/`, then building the Astro static site.

## Environment Variables

Copy `.env.example` to `.env` and provide every value listed below. All variables are required for a full working deployment.

### `PUBLIC_CLOUDINARY_CLOUD_NAME`

Your Cloudinary cloud name. Found in the [Cloudinary Dashboard](https://cloudinary.com/console) under **Settings → Account → Cloud name**.

Used to construct image delivery URLs. No SDK is involved — all image URLs are assembled as plain strings using this value.

### `RESEND_API_KEY`

Your Resend secret API key. Found in the [Resend Dashboard](https://resend.com/api-keys) under **API Keys → Create API Key**.

Keep this secret. It is only used server-side in the `/api/contact` endpoint and is never exposed to the browser.

### `RESEND_FROM_EMAIL`

The verified sender address for outgoing contact form emails — for example, `noreply@yaelcohen.photography`.

**Important:** This address must belong to a domain you have verified in Resend. Go to [Resend → Domains](https://resend.com/domains), add your domain, and follow the DNS verification steps. Sending from an unverified domain will cause all API calls to fail with a 403 error.

### `RESEND_RECIPIENT_EMAIL`

The email address that receives contact form submissions from visitors — for example, `hello@yaelcohen.photography`. This does not need to be verified in Resend.

### `TINA_CLIENT_ID`

Your Tina Cloud project Client ID. Found in the [Tina Cloud Dashboard](https://app.tina.io) under your project → **Overview → Client ID**.

### `TINA_TOKEN`

Your Tina Cloud read/write token. Found in the Tina Cloud Dashboard under your project → **Tokens → Add Token**. Keep this secret — it grants write access to your content repository.

### `PUBLIC_TINA_CLIENT_ID`

Set this to the same value as `TINA_CLIENT_ID`. Astro only exposes environment variables to the browser if they start with `PUBLIC_`. The Tina admin UI bundle requires this value at runtime to authenticate with Tina Cloud.

## Configuring Tina Cloud

Tina Cloud connects your CMS to your Git repository so that content edits made in the admin panel are committed directly as JSON file changes.

1. Go to [app.tina.io](https://app.tina.io) and sign in (GitHub login recommended).
2. Click **New Project** and connect your GitHub repository.
3. Copy the **Client ID** from the project Overview page — this is your `TINA_CLIENT_ID` and `PUBLIC_TINA_CLIENT_ID`.
4. Go to **Tokens** and create a new token with read/write access — this is your `TINA_TOKEN`.
5. Under **Configuration**, set the **Branch** to `main` (or whichever branch you deploy from).
6. Tina Cloud will automatically index your content on first deploy. The schema in `tina/config.ts` defines all collections and fields.

To access the visual CMS editor in production, navigate to `https://yaelcohen.photography/admin`. You must be logged in to Tina Cloud for edits to save.

## Connecting Resend and Verifying a Domain

1. Create a free account at [resend.com](https://resend.com).
2. Go to **Domains → Add Domain** and enter the domain you will send from (e.g. `yaelcohen.photography`).
3. Add the DNS records shown (SPF, DKIM, and optionally DMARC) to your domain registrar. Verification typically takes a few minutes to a few hours.
4. Once the domain shows **Verified**, go to **API Keys → Create API Key** and copy the key into `RESEND_API_KEY`.
5. Set `RESEND_FROM_EMAIL` to any address at your verified domain (e.g. `noreply@yaelcohen.photography`).
6. Set `RESEND_RECIPIENT_EMAIL` to the address where you want to receive inquiry notifications.

## Setting Up Cloudinary

1. Create a free account at [cloudinary.com](https://cloudinary.com).
2. From the [Cloudinary Console](https://cloudinary.com/console), find your **Cloud name** in the top left. Copy it into `PUBLIC_CLOUDINARY_CLOUD_NAME`.
3. Upload your photographer's images to Cloudinary. Note the **Public ID** of each uploaded image (shown in the Media Library).
4. Replace the placeholder Cloudinary Public IDs in the gallery and about page content:
   - `src/content/gallery/photo-01.json` through `photo-08.json` — update the `cloudinaryPublicId` field in each file, or use the Tina CMS admin to update them visually.
   - `src/content/aboutPage/content.json` — update `profileImageCloudinaryId` with the Public ID of the photographer's portrait.
5. The site uses Cloudinary's automatic format (`f_auto`) and quality (`q_auto`) transforms — no manual optimisation is needed.

**Note on development:** The seed content uses Cloudinary's public sample library IDs (e.g. `samples/people/jazz`). These render real images in development without any account setup, so you can preview the site immediately after cloning.

## Deploying to Vercel

1. Push your repository to GitHub (or GitLab / Bitbucket).
2. Go to [vercel.com](https://vercel.com), click **Add New Project**, and import your repository.
3. Vercel auto-detects Astro — no framework preset configuration is required.
4. Before deploying, go to **Settings → Environment Variables** in the Vercel project dashboard and add all seven environment variables from your `.env` file:
   - `PUBLIC_CLOUDINARY_CLOUD_NAME`
   - `RESEND_API_KEY`
   - `RESEND_FROM_EMAIL`
   - `RESEND_RECIPIENT_EMAIL`
   - `TINA_CLIENT_ID`
   - `TINA_TOKEN`
   - `PUBLIC_TINA_CLIENT_ID`
5. Click **Deploy**. Vercel runs `tinacms build && astro build` automatically.
6. Once deployed, set your custom domain (`yaelcohen.photography`) in Vercel → **Settings → Domains** and follow the DNS instructions.

The `/api/contact` endpoint is automatically deployed as a Vercel serverless function because it exports `prerender = false`. All other pages are pre-rendered static HTML served from Vercel's CDN.

## Accessing the CMS Admin in Production

Once deployed, the Tina CMS visual editor is accessible at:

```
https://yaelcohen.photography/admin
```

The `/admin` path is served via the rewrite rule in `vercel.json` (`/admin` → `/admin/index.html`).

To make edits:
1. Navigate to `https://yaelcohen.photography/admin`.
2. Click **Sign in with Tina Cloud** and authenticate with your Tina account.
3. Select a content type from the sidebar (Gallery Photos, Testimonials, Site Settings, About Page, Pages SEO).
4. Make your changes and click **Save**. Tina commits the updated JSON files directly to your GitHub repository.
5. Vercel detects the commit and automatically rebuilds and redeploys the site within seconds.

## i18n Architecture

All user-visible strings live in `src/i18n/translations.ts`. No string literals appear in component or page files. The translation utility `t(locale, key)` is imported into every component that needs text. Hebrew pages are served at `/` (no prefix) and English pages at `/en/...`.

RTL layout is handled natively via CSS logical properties throughout — `padding-inline`, `margin-block`, `inset-inline-start`, etc. The `dir="rtl"` attribute is set on `<html>` for Hebrew pages via `BaseLayout.astro`.
